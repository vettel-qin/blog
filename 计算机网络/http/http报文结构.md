HTTP报文结构是header + body，具体结构是起始行 + 头部 + 空行 + 实体

# 起始行
## 请求报文
由方法 + 路径 + HTTP版本 组成
```
GET /index HTTP/1.1
```
## 响应报文
由HTTP版本 + 状态码 + 原因 组成
```
HTTP/1.1 200 OK
```

# 头部
不管是请求头部还是响应头部，其中的字段是相当多的。例如：

请求头部
```
Host: 请求的主机名
User-Agent: 产生请求的浏览器类型。
accept: 请求头用来告知服务器(客户端)可以处理的内容类型。
```

响应头部
```
Content-Type: 告诉客户端实际返回的内容的内容类型。
Cache-Control: 用于缓存控制
```

# 空行
用来区分开头部和实体
如果在头部中间故意加一个空行会怎么样？
答应是空行后的内容全部被视为实体

# 实体
具体的数据，就是我们常说的body部分
